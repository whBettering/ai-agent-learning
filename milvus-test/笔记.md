# Milvus RAG 项目笔记

## 一、项目概览

用 **Milvus 向量数据库** 替代 `MemoryVectorStore`，实现持久化的 AI 日记问答系统。

```
insert.mjs          query.mjs           rag.mjs
（数据入库）         （纯检索）           （完整 RAG）
    │                   │                   │
    ▼                   ▼                   ▼
  创建集合           生成向量            生成向量
  生成向量           Milvus 搜索         Milvus 搜索
  插入 Milvus        打印结果            拼 prompt
                                         LLM 生成答案
```

## 三、三个文件详解

### 1. insert.mjs — 数据入库

**做了什么：**

| 步骤 | 代码关键点 |
|------|------|
| 连接 Milvus | `new MilvusClient({ address: 'localhost:19530' })` |
| 创建 Collection | 定义字段：`id`, `vector`, `content`, `date`, `mood`, `tags` |
| 创建索引 | `IVF_FLAT` + `COSINE` 相似度 |
| 加载 Collection | 加载到内存才能搜索 |
| 批量插入 | 5 篇日记 → 调 embedding API → 向量 + 元数据写入 Milvus |

**Collection 字段设计：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | VarChar(50) | 主键 |
| `vector` | FloatVector(1024) | 文本嵌入向量 |
| `content` | VarChar(5000) | 日记正文 |
| `date` | VarChar(50) | 日期 |
| `mood` | VarChar(50) | 心情标签 |
| `tags` | Array(VarChar) | 标签列表 |

---

### 2. query.mjs — 纯检索

只做两件事：
1. 把问题转成向量
2. 在 Milvus 中搜索 top-2 最相似日记，打印原始结果

**用途**：调试检索效果，验证向量搜索是否准确。

---

### 3. rag.mjs — 完整 RAG

比 query.mjs 多了：
- 导入 `ChatOpenAI`（LLM）
- 拼装 prompt（含角色设定、上下文、回答要求）
- LLM 生成自然语言回答

## 四、核心概念

### Milvus 基础概念映射

| Milvus | 类比关系型数据库 | 说明 |
|------|------|------|
| Collection | Table | 一张表 |
| Field | Column | 字段 |
| Entity | Row | 一条记录 |
| Index | Index | 加速向量搜索 |
| Partition | Partition | 数据分区 |

### 索引类型

```
IVF_FLAT：倒排索引 + 暴力搜索
  - 先把向量聚类成 nlist 个簇
  - 搜索时只查最近的几个簇
  - 牺牲一点精度，换来速度
```

### 相似度度量

```
COSINE：余弦相似度（本项目使用）
  - 值域 [-1, 1]，越接近 1 越相似
  - 关注方向而非长度，适合文本语义比较
```

## 五、与之前 RAG 的对比

| | hello-rag.mjs | milvus-test |
|------|------|------|
| 向量库 | `MemoryVectorStore`（内存） | **Milvus**（持久化） |
| 数据入库 | 每次启动执行 | 一次 `insert.mjs`，持久保存 |
| 规模 | 适合 demo/小数据 | **亿级向量** |
| 部署 | 无需额外服务 | 需启动 Milvus 服务 |
| 检索方式 | LangChain 封装 | **直接用 Milvus SDK** |
