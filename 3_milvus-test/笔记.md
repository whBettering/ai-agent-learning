# 电子书 RAG 项目（《活着》）

## 一、项目概览

基于 **Milvus 向量数据库**，实现《活着》电子书的智能问答系统——输入 EPUB，AI 就能回答书中内容。

```
ebook-writer.mjs          ebook-query.mjs          ebook-reader-rag.mjs
（解析 EPUB → 入库）       （纯检索调试）            （完整 RAG 问答）
      │                         │                         │
      ▼                         ▼                         ▼
 EPubLoader 解析          生成查询向量              生成查询向量
 按章节拆分               Milvus 检索               Milvus 检索
 TextSplitter 再切分       打印原始结果              拼 prompt
 生成 Embedding                                      LLM 生成答案
 写入 Milvus
```

## 三、ebook-writer.mjs — EPUB 解析与入库

### 核心流程

```
活着.epub
  │  EPubLoader({ splitChapters: true })
  ▼
按章节拆分（10+ 章）
  │  RecursiveCharacterTextSplitter({ chunkSize: 500, chunkOverlap: 50 })
  ▼
每章切成 N 个片段
  │  调 embedding API → 向量
  ▼
逐章流式插入 Milvus
```

### 关键点

**EPubLoader — 自动解析 EPUB：**
- EPUB 本质是 **zip 压缩的 HTML 文件集合**
- `EPubLoader` 解压 → 提 HTML → 转纯文本
- `splitChapters: true`：自动检测章节边界

**两级拆分：**

| 级别 | 工具 | 粒度 |
|------|------|------|
| 第一级 | `EPubLoader.splitChapters` | 按章节 |
| 第二级 | `RecursiveCharacterTextSplitter` | 每章再切到 500 字符 |

**流式入库（核心优化）：**

```js
for (let i = 0; i < documents.length; i++) {
  const chunks = await textSplitter.splitText(documents[i].pageContent);
  await insertChunksBatch(chunks, bookId, i + 1);  // 逐章处理，立即插入
}
```

整本书几百个片段，逐章处理避免内存溢出。

**结构化 ID：** `bookId_chapterNum_index`（如 `1_3_5` → 第 3 章第 5 个片段），方便定位原文。

**书名提取：** `parse('./活着.epub').name` → `"活着"`

## 五、ebook-reader-rag.mjs — 完整 RAG 问答

在 ebook-query 的基础上，增加了 LLM 生成环节：

```js
// 检索 → 拼 prompt → LLM 回答
const context = retrievedContent.map(item => `[片段] ${item.content}`).join('\n');
const prompt = `你是《活着》小说助手。基于以下片段回答：${context}。问题：${question}`;
const response = await model.invoke(prompt);
```

## 六、Collection 字段设计

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | VarChar(100) | `bookId_chapterNum_index` |
| `book_id` | VarChar(100) | 书籍唯一标识 |
| `book_name` | VarChar(200) | 书名（多书支持） |
| `chapter_num` | Int32 | 章节号 |
| `index` | Int32 | 片段在章节内的序号 |
| `content` | VarChar(10000) | 文本内容 |
| `vector` | FloatVector(1024) | 嵌入向量 |

## 九、与之前 MemoryVectorStore 的关键区别

| | 之前（hello-rag） | 本项目 |
|------|------|------|
| 向量库 | `MemoryVectorStore`（内存） | **Milvus**（持久化） |
| 数据导入 | 每次启动执行 | 一次 `ebook-writer`，持久保存 |
| 规模 | demo 级 | **亿级向量** |
| 部署 | 无需额外服务 | 需启动 `milvus-standalone` |
| 数据源 | 硬编码文本 | **真实 EPUB 文件** |
