# LangChain Runnable 学习笔记

本章节主要介绍 LangChain 中的 Runnable 机制。Runnable 可以理解为“可执行的处理单元”，它不仅能处理单个输入，还能把多个步骤串联成一条链，形成更灵活的工作流。

---

## 一、什么是 Runnable

在 LangChain 里，Runnable 是一个统一抽象，表示一种可运行、可组合的处理节点。它通常具备以下特点：

- 支持 `invoke()` 直接执行
- 可以串联成流水线
- 可以嵌套组合，形成复杂流程
- 适合做 prompt、模型调用、数据处理、条件分支等步骤编排

它是 LangChain 中“链式编排”的核心基础。

---

## 二、从普通写法到 Runnable 写法

### 1. 传统写法

在 [src/before.mjs](src/before.mjs) 中，流程是手动执行三步：

1. 用 `PromptTemplate` 生成 prompt
2. 调用模型生成结果
3. 用 `outputParser` 解析结果

这种写法清晰，但步骤之间是“手写串联”的，不够模块化。

### 2. Runnable 写法

在 [src/runnable.mjs](src/runnable.mjs) 中，使用了 `RunnableSequence`：

```js
const chain = RunnableSequence.from([
  promptTemplate,
  model,
  outputParser
]);
```

这种写法把流程变成了一条“链”，语义更清晰，也更适合扩展。

> 核心理解：把“步骤”变成“节点”，再把节点组合起来。

## 三、常见 Runnable 类型

### 1. RunnableLambda

`RunnableLambda` 用于把普通函数包装成 Runnable。

它适合：

- 数据转换
- 数学运算
- 文本清洗
- 自定义业务逻辑

示例文件：[src/runnables/RunnableLambda.mjs](src/runnables/RunnableLambda.mjs)

```js
const addOne = RunnableLambda.from((input) => input + 1);
```

它的作用非常简单：把函数变成可被链式调用的节点。

---

### 2. RunnableSequence

`RunnableSequence` 是最常见的链式编排方式。

它把多个 Runnable 按顺序串起来，前一个节点的输出成为下一个节点的输入。

示例文件：[src/runnable.mjs](src/runnable.mjs)

```js
const chain = RunnableSequence.from([
  promptTemplate,
  model,
  outputParser
]);
```

使用场景：

- prompt 生成 → 模型调用 → 输出解析
- 数据清洗 → 处理 → 结果返回

---

### 3. RunnableMap

`RunnableMap` 允许同时并行执行多个 Runnable，并把结果汇总成对象。

示例文件：[src/runnables/RunnableMap.mjs](src/runnables/RunnableMap.mjs)

它适合以下场景：

- 同时生成多个子结果
- 同时做多个 prompt 格式化
- 同时计算多个字段

例如同时计算：

- 加法
- 乘法
- 平方
- 问候语
- 天气提示

这体现了“并发/并行子任务”的思路。

---

### 4. RunnablePick

`RunnablePick` 用来从输入对象中选择一部分字段，输出一个更精简的对象。

示例文件：[src/runnables/RunnablePick.mjs](src/runnables/RunnablePick.mjs)

```js
new RunnablePick(["name", "email", "fullInfo"])
```

适合场景：

- 过滤多余字段
- 提取关键信息
- 只把需要的数据传给下一步

---

### 5. RunnablePassthrough

`RunnablePassthrough` 的作用是“原样传递输入”，常用于保留原始输入并向下游继续传递。

示例文件：[src/runnables/RunnablePassthrough.mjs](src/runnables/RunnablePassthrough.mjs)

它特别适合：

- 保留原始输入
- 在处理过程中同时输出“原始值”和“处理后结果”
- 在链式流程中保持上下文不丢失

---

### 6. RunnableBranch

`RunnableBranch` 用来做条件分支判断。

示例文件：[src/runnables/RunnableBranch.mjs](src/runnables/RunnableBranch.mjs)

它的逻辑是：

- 先判断条件
- 条件满足就走对应分支
- 否则走默认分支

适合场景：

- 正负数判断
- 用户角色判断
- 多种输入类型的处理分流

---

### 7. RunnableEach

`RunnableEach` 用于对数组中的每个元素依次执行同一条处理链。

示例文件：[src/runnables/RunnableEach.mjs](src/runnables/RunnableEach.mjs)

它适合：

- 批量文本处理
- 批量数据清洗
- 批量翻译或分类

核心点是：把“单条处理逻辑”变成“批量处理逻辑”。

---

## 四、可观测性：Callbacks

在 [src/runnables/RunnableWithCallbacks.mjs](src/runnables/RunnableWithCallbacks.mjs) 中，使用了 callbacks 来观察链的执行过程。

它可以监听：

- 链开始执行
- 链成功结束
- 链执行报错

这对于调试流程非常有帮助。

### 为什么重要

在复杂链中，很多问题并不是“最终结果错了”，而是中间某一步输出异常。Callbacks 能帮助你快速定位问题。

---

## 五、运行时配置：withConfig

在 [src/runnables/RunnableWithConfig.mjs](src/runnables/RunnableWithConfig.mjs) 中，使用了 `withConfig()` 给整个链绑定统一配置。

它可以配置：

- `tags`：给链打标签
- `metadata`：附加元数据
- `configurable`：注入运行时参数

例如：

- 当前用户 ID
- 用户角色
- 语言环境（locale）

这使得同一套链可以在不同上下文下运行，而不需要复制一份业务逻辑。

### 关键点

`configurable` 非常适合做：

- 多租户场景
- 权限控制
- 多语言渲染
- 环境差异化配置

---

## 六、容错与重试

### 1. withFallbacks

在 [src/runnables/RunnableWithFallbacks.mjs](src/runnables/RunnableWithFallbacks.mjs) 中，使用了 `withFallbacks()`。

它的思想是：

- 先尝试主服务
- 如果主服务失败，自动切换到备用服务

例如：

- Premium 翻译服务挂了
- 切换到 Standard 服务
- 再不行就用本地词典兜底

这在真实工程中非常实用，尤其是外部 API 依赖场景。

### 2. withRetry

在 [src/runnables/RunnableWithRetry.mjs](src/runnables/RunnableWithRetry.mjs) 中，使用了 `withRetry()`。

它的作用是：

- 对失败的节点自动重试
- 降低偶发错误带来的影响

例如：

- 网络抖动
- 临时限流
- 随机失败

在生产环境里，重试机制往往是稳定性的重要一环。

---

## 七、消息历史：RunnableWithMessageHistory

在 [src/runnables/RunnableWithMessageHistory.mjs](src/runnables/RunnableWithMessageHistory.mjs) 中，介绍了一个更贴近聊天场景的能力：`RunnableWithMessageHistory`。

它的作用是：

- 记住历史对话
- 按会话 ID 保存消息
- 让后续问答能够基于上下文回答

这非常适合：

- AI 聊天机器人
- 多轮对话助手
- 需要连续上下文的应用

### 核心思路

- 第一次输入：保存用户信息
- 第二次输入：模型根据历史上下文继续回答

这就是“有状态对话链”的基础。

---

## 八、这一章节的学习重点

这一章节最值得掌握的不是某一个单独的类，而是“Runnable 编排思想”。

### 你可以这样理解

- `RunnableLambda`：把函数变成可组合节点
- `RunnableSequence`：把节点串成链
- `RunnableMap`：把节点并行组织
- `RunnableBranch`：让流程根据条件分流
- `RunnableEach`：让单条逻辑批量处理
- `withConfig`：给链注入运行时上下文
- `withFallbacks` / `withRetry`：提升系统可靠性
- `RunnableWithMessageHistory`：让链具备对话记忆

---

## 九、实战感悟

如果把 LangChain 想象成一个“可编排的 AI 工作流引擎”，那么 Runnable 就是它最基础的“零件”。

真正有价值的不是单独记住某个类，而是理解：

- 复杂任务如何拆成多个步骤
- 各步骤如何顺序执行、条件分支、并行执行
- 中间结果如何传递和控制
- 如何让链具备可观测性、容错性和状态记忆

这也是后面学习 Agent、工具调用、记忆系统等内容的基础。

---

## 十、结论

本章节的核心结论可以概括为一句话：

> Runnable 让 LangChain 的工作流从“手写流程”升级为“可组合、可重用、可观测、可扩展的流程编排”。
