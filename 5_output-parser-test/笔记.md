# 5_output-parser-test 暂存区笔记

该项目主要演示 LangChain + OpenAI 的“输出解析器（output parser）”用法，包含几类不同的结构化输出示例。

## 主要依赖

- `@langchain/core`
- `@langchain/openai`
- `dotenv`
- `zod`
- `zod-to-json-schema`
- `chalk`
- `mysql2`

这些依赖反映了项目既有 OpenAI 调用，也有结构化数据定义、JSON Schema、流式处理相关示例。

## 关键文件说明

### `src/normal.mjs`
- 最基础示例。
- 直接用 `ChatOpenAI.invoke(prompt)` 获取文本响应。
- 通过 `JSON.parse(response.content)` 解析返回字符串为对象。
- 不使用 LangChain 的输出解析器，仅靠提示要求 JSON。

### `src/json-output-parser.mjs`
- 使用 LangChain 内置的 `JsonOutputParser`。
- 通过 `parser.getFormatInstructions()` 将 parser 的格式说明插入 prompt。
- 这一步比普通提示更严格，减少模型输出额外文本、代码块、注释等情况。
- 最终用 `parser.parse(response.content)` 自动解析 JSON。
- 这份代码里已补充注释，解释了“自然语言提示”和“parser 格式说明”的区别。

### `src/structured-output-parser.mjs`
- 使用 `StructuredOutputParser.fromNamesAndDescriptions(...)` 定义字段名和描述。
- prompt 中写入 `parser.getFormatInstructions()`，让模型按照结构化输出返回。
- `parser.parse(response.content)` 解析模型响应为对象。
- 适合输出字段已知、且希望模型严格遵守指定字段结构的场景。

### `src/xml-output-parser.mjs`
- 使用 `XMLOutputParser`。
- prompt 中同样插入 `parser.getFormatInstructions()`。
- 用于让模型返回 XML 格式数据，并由 parser 自动解析。
- 适合需要 XML 数据交换格式的场景。

### `src/structured-json-schema.mjs`
- 演示使用 `zod` 定义 schema，然后通过 `zod-to-json-schema` 转成 JSON Schema。
- 直接把 JSON Schema 传给模型的 `response_format`，让模型输出纯 JSON。
- 这是更“底层”的方式，跳过了 `OutputParser` 的 wrapper，直接使用模型原生的 JSON Schema 输出能力。

### `src/stream-with-structured-output.mjs`
- 通过 `model.withStructuredOutput(schema)` 创建支持结构化输出的模型。
- 调用 `structuredModel.stream(prompt)`，获取流式结构化输出。
- 展示如何逐块接收数据，并在最后汇总为结构化结果。
- 适合需要实时接收、或大结果需要流式处理的场景。

### `src/stream-tool-calls-parser.mjs`
- 使用 `JsonOutputToolsParser` 和工具调用（tool call）方式。
- 定义工具 schema，并用 `model.bindTools([ ... ])` 绑定到模型。
- `chain.stream(...)` 开启流式工具调用解析。
- 展示了如何实时打印工具调用参数流，并最终得到结构化结果。
- 适合需要以“工具调用”方式与模型协同时提取结构化数据的场景。

## 核心结论

- 单纯写 `请以 JSON 返回` 不一定足够。
- `parser.getFormatInstructions()` 是给模型更明确的格式说明，能显著提高解析成功率。
- LangChain 提供多种 parser：JSON、Structured、XML、tools parser、以及直接 JSON Schema 输出。
- 流式输出与结构化输出可以组合使用，适合实时数据接收与大模型长响应场景。

## 使用建议

- 如果只是简单 JSON，先尝试 `JsonOutputParser`，并把 `parser.getFormatInstructions()` 插入 prompt。
- 如果字段结构明确，优先使用 `StructuredOutputParser` 或 `withStructuredOutput(schema)`。
- 如果需要 XML 或工具调用，使用对应的 `XMLOutputParser` 或 `JsonOutputToolsParser`。
- 需要更强校验时，可结合 `zod` 定义 schema，并转成 JSON Schema。
