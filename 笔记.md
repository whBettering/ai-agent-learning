# LangChain Prompt Templates 学习笔记

本笔记基于 `6_prompt-template-test` 目录下的 13 个示例文件整理，涵盖 LangChain 中 Prompt 模板的核心用法。

---

## 一、基础概念速查

### 1. PromptTemplate

- 用于纯文本 prompt 的模板。
- `fromTemplate(templateStr)`：静态工厂方法（快捷方式），自动从模板字符串中提取 `{变量名}`，返回 `PromptTemplate` 实例。
- `format(values)`：实例方法，将模板中的 `{变量名}` 替换为实际值，返回拼接后的完整 prompt 字符串。
- 同一实例可多次调用 `format`，传入不同数据，复用模板结构。

> 参考文件：`prompt-template1.mjs`

### 2. ChatPromptTemplate

- 用于聊天场景的模板，生成消息数组（system / human / ai），而不是纯文本。
- `fromMessages([...])`：接收消息模板数组。

> 参考文件：`chat-prompt-template.mjs`

### 3. SystemMessagePromptTemplate / HumanMessagePromptTemplate

- 对 system / human 角色的消息单独声明，再组合成 `ChatPromptTemplate`。
- 优点：每个角色的模板可独立复用。

> 参考文件：`chat-prompt-template2.mjs`

### 4. MessagesPlaceholder

- 在 `ChatPromptTemplate` 中预留一个位置，运行时插入历史对话消息。
- 用于多轮对话场景。

> 参考文件：`messages-placeholder.mjs`

## 二、Few-Shot Prompt（少样本提示）

核心思路：给模型看几条"输入 → 期望输出"的示例，让它学习风格和结构。

### 1. FewShotPromptTemplate（纯文本版）

需要三个要素：

- `examplePrompt`：定义单条示例的格式模板
- `examples`：示例数据数组
- `prefix` / `suffix`：包裹示例的前后缀

`format({})` 会把 示例 + 前后缀 拼接成完整的 prompt。

> 参考文件：`fewshot-prompt-template.mjs`

### 2. FewShotChatMessagePromptTemplate（聊天版）

- 每条示例定义为 `{ input, output }`，映射为 human → ai 的消息对。
- 嵌入 `ChatPromptTemplate` 作为一个消息块。

> 参考文件：`fewshot-chat-prompt-template.mjs`

---

## 三、Example Selector（示例选择器）

**问题**：示例太多时，全部塞进 prompt 会超 token 上限。  
**解决**：用 Selector 根据当前输入，自动筛选最合适的几条示例。

### 1. LengthBasedExampleSelector（基于长度）

- 按字符长度从短到长选示例，直到总长度超 `maxLength` 停止。
- 输入越长（需求越复杂），能塞进的示例越少。

> 参考文件：`example-selector1.mjs`

### 2. SemanticSimilarityExampleSelector（基于语义相似度）

- 用向量相似度匹配最相关的 k 条示例。
- 需要配合 Milvus 向量库使用：
  - `weekly-report-examples-writer-milvus.mjs`：先将示例写入 Milvus
  - `example-selector2.mjs`：从 Milvus 中检索语义最相近的示例

> 参考文件：`example-selector2.mjs`、`weekly-report-examples-writer-milvus.mjs`

## 四、PipelinePromptTemplate（流水线模板）

核心思路：把 prompt 拆成多个独立模块（人设、背景、任务、格式），每个模块独立定义、独立复用，最后用 `PipelinePromptTemplate` 组合。

### 1. 基础 Pipeline

定义 4 个模块：

| 模块 | 内容 |
|------|------|
| `personaPrompt` | 人设风格 |
| `contextPrompt` | 公司/团队背景 |
| `taskPrompt` | 任务与数据 |
| `formatPrompt` | 输出格式要求 |

用 `PipelinePromptTemplate` 拼接，通过 `{persona_block}` 等占位符注入。

> 参考文件：`pipeline-prompt-template.mjs`

### 2. 跨场景复用模块

- 从 `pipeline-prompt-template.mjs` 导出 `personaPrompt` 和 `contextPrompt`。
- 在新的"季度 OKR 回顾邮件"场景中，复用这两个模块 + 自定义 task/format 模块。
- 证明"模块化拆分"的复用价值。

> 参考文件：`pipeline-prompt-template2.mjs`

### 3. Pipeline + ChatPromptTemplate

- 将 Pipeline 的最终输出作为 `ChatPromptTemplate` 的一部分。
- Pipeline 负责拼接文本块，`ChatPromptTemplate` 负责组装成消息数组。

> 参考文件：`pipeline-prompt-template3.mjs`

## 五、Partial 变量（部分填充）

- `partial({...})` 方法：预先固定部分变量值，返回一个新的 `PromptTemplate` 实例。
- 后续 `format` 时只需填入剩余变量。
- 场景：公司名称、价值观等固定信息先 `partial`，再多次用不同数据 `format`。

> 参考文件：`partial.mjs`（复用 `pipeline-prompt-template.mjs` 的 `pipelinePrompt`）

## 七、概念关系图

```
PromptTemplate ─── 纯文本模板
  ├── fromTemplate() ─── 静态工厂，创建实例
  ├── format()     ─── 实例方法，填充变量
  └── partial()    ─── 实例方法，部分填充

ChatPromptTemplate ─── 聊天消息模板
  ├── fromMessages([...])
  ├── SystemMessagePromptTemplate
  ├── HumanMessagePromptTemplate
  └── MessagesPlaceholder ─── 历史对话占位

FewShotPromptTemplate ─── 少样本模板（纯文本）
FewShotChatMessagePromptTemplate ─── 少样本模板（聊天）

ExampleSelector ─── 示例选择器
  ├── LengthBasedExampleSelector ─── 按长度截断
  └── SemanticSimilarityExampleSelector ─── 按语义匹配（需要向量库）

PipelinePromptTemplate ─── 流水线组合模板
  ├── pipelinePrompts[] ─── 子模块列表
  └── finalPrompt ─── 最终拼接模板
```
