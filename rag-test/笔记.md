# hello-rag.mjs 代码笔记

## 一、整体架构：RAG（检索增强生成）

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  文档入库  │ ──▶ │  向量检索  │ ──▶ │  LLM 生成 │
│ (Embedding)│     │ (Retrieve)│     │ (Generate)│
└──────────┘     └──────────┘     └──────────┘
```

---

## 二、使用的模型

| 用途 | 模型 | 来源 |
|------|------|------|
| 对话生成 | `qwen-max` | 阿里云 DashScope（OpenAI 兼容接口） |
| 文本嵌入 | `text-embedding-v3` | 同上（建议升级到 `text-embedding-v4`） |

> 代码通过 `@langchain/openai` 调用，但 `baseURL` 指向了 `dashscope.aliyuncs.com/compatible-mode/v1`，实际调用的是阿里云百炼平台。

## 三、核心流程拆解

### 1. 初始化模型实例

```js
// 对话模型
const model = new ChatOpenAI({ temperature: 0, model: 'qwen-max', ... });

// 嵌入模型
const embeddings = new OpenAIEmbeddings({ model: 'text-embedding-v3', ... });
```

### 2. 准备知识库文档（7 篇小故事）

```js
const documents = [
  new Document({
    pageContent: '文档正文...',
    metadata: { chapter: 1, character: '光光', type: '角色介绍', mood: '活泼' }
  }),
  // ... 共 7 篇
];
```

`metadata` 用于后续过滤和展示，**不参与向量检索**。

### 3. 文档向量化 + 入向量库

```js
const vectorStore = await MemoryVectorStore.fromDocuments(documents, embeddings);
```

- 调用嵌入模型，把每篇文档转成高维向量
- `MemoryVectorStore`：向量存内存，重启即丢失，仅适合 demo

### 4. 创建检索器

```js
const retriever = vectorStore.asRetriever({ k: 3 });
```

`k: 3` 表示每次检索返回最相似的 3 篇文档。

### 5. 检索 + 生成答案

```js
// 用户问题
const question = "东东和光光是怎么成为朋友的？";

// 向量检索：把问题也转成向量，在库中找最相似的文档
const retrievedDocs = await retriever.invoke(question);

// 组装 prompt：检索到的文档作为上下文
const prompt = `基于以下故事片段回答问题...

故事片段: ${context}
问题: ${question}`;

// 交给 LLM 生成最终答案
const response = await model.invoke(prompt);
```

## 四、关键技术点

### Embedding（嵌入）的作用

| | 文本 | 向量表示 |
|------|------|------|
| 文档 A | "光光擅长踢足球" | `[0.12, -0.34, 0.78, ...]` |
| 问题 Q | "东东和光光怎么认识的" | `[0.11, -0.31, 0.75, ...]` |

> 语义越相似，向量之间的余弦距离越小 → 检索越精准。

### MemoryVectorStore vs 生产环境

| 方案 | 特点 |
|------|------|
| `MemoryVectorStore` | 内存存储，每次重启重建索引 |
| `Chroma` / `Pinecone` / `Milvus` | 持久化，支持海量数据 |

### 相似度评分

代码额外调用了 `similaritySearchWithScore` 获取评分：
- 返回的是**距离**（越小越相似）
- 用 `1 - score` 转换为相似度

## 五、改进建议

1. **嵌入模型升级**：`text-embedding-v3` → `text-embedding-v4`（v3 已不在百炼推荐列表）
2. **持久化向量库**：替换 `MemoryVectorStore` 为 `Chroma` 或 `FAISS`，避免每次重启重建索引
3. **批量缓存**：如果文档不会变，缓存 embedding 结果，避免重复调用 API
4. **元数据过滤**：检索时可按 `mood`、`type` 等 metadata 先过滤，提高精度
